# arvern Trace Viewer

Interactive Streamlit application for analyzing arvern execution traces.

## Quick start

```bash
# Install dependencies (once)
pip3 install -r requirements.txt

# Launch from this directory
streamlit run app.py

# Or launch from anywhere
streamlit run /path/to/trace_viewer/app.py
```

## Views

| View | Description |
|------|-------------|
| **Overview** | Trace metadata, global KPIs, instruction category pie |
| **Pipeline** | CPI/IPC, stall breakdown by cause, CPI per category |
| **Instruction Mix** | Category/mnemonic frequency, extension utilization |
| **Branches** | Taken/forward/backward %, register pair analysis, transition matrix; JALR/JAL call-return stats |
| **Memory** | Load/store ratio, access sizes, stride histogram, load-to-use distance |
| **Registers** | 4×8 write/read heatmaps, reuse distance, top registers |
| **Hot Code** | Most executed PC addresses, execution density |
| **Dependencies** | Producer→consumer distance, common RAW chains |
| **N-grams** | Most common bigrams/trigrams, macro-fusion candidates |
| **Loops** | Back-edge detection, trip count histogram, loop body sizes |
| **Compare** | Side-by-side KPIs, category diff, mnemonic frequency diff |

## Trace files

Traces are generated by `save_trace.py` (plain tests) or `store_benchmark.py`
(benchmarks). Both formats are supported:

- `.log` — plain text
- `.log.zst` — Zstandard-compressed (load via file uploader or path input)
